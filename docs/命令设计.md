# siyuan-cli 命令设计文档

## 设计原则

1. **资源导向**：命令围绕资源（notebook / document / block / asset / tag）组织
2. **动词+资源**：`<动词> <资源>` 风格，与现有命令一致
3. **路径参数**：支持人类可读路径（如 `java面试/AI/index`）和文档 ID 两种方式
4. **输出格式**：全局 `--format json` + 各命令 `--output, -o` 导出文件
5. **管道友好**：`get` 类命令输出纯文本/JSON，可配合管道使用

## 通用模式

### 输出格式

所有命令统一支持：
- **默认**：格式化表格/树形（终端友好）
- **`--format json`**（全局）：JSON 输出
- **`--output, -o <file>`**：导出到文件（自动使用 JSON 格式）

### 笔记本匹配

所有需要指定笔记本的命令，统一支持：
- 笔记本名称（模糊匹配）
- 笔记本 ID（精确匹配）

复用 `FindNotebook()` 函数。

### 文档路径

所有需要指定文档的命令，统一支持：
- 人类可读路径（如 `java面试/AI/index`）
- 文档 ID

复用 `resolveDocPath()` 函数。

### 错误处理

- 无效参数 → 友好提示 + 正确用法示例
- API 错误 → 中文错误信息 + 建议操作
- 连接失败 → 提示运行 `auth login`
